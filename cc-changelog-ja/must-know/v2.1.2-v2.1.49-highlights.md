# Claude Code v2.1.2〜v2.1.49 重要アップデートまとめ

> cc-changelog-ja.md の約40バージョン・537行分から、知っておくべき重要な変更をピックアップ。
> 各項目の `(vX.X.XX)` は対応するバージョン番号。詳細は cc-changelog-ja.md を参照。

---

## 1. 新機能・ワークフロー革新

### Agent Teams（エージェントチーム）
- マルチエージェントコラボレーション向けのリサーチプレビュー版を追加 (v2.1.32)
  - `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` の設定が必要（トークン集約型機能）
  - tmux でのチームメイトセッションがメッセージの送受信を行えるように (v2.1.33)
  - `TeammateIdle` および `TaskCompleted` フックイベントを追加 (v2.1.33)
  - Bedrock、Vertex、Foundry で API プロバイダー環境変数を tmux 生成プロセスに伝播 (v2.1.45)

### Worktree 統合
- Claude を隔離された git worktree で起動する `--worktree`（`-w`）フラグを追加 (v2.1.49)
- サブエージェントが一時的な git worktree で作業するための `isolation: "worktree"` をサポート (v2.1.49)

### メモリ自動記録
- Claude が作業中にメモリを自動的に記録・想起するように (v2.1.32)
- エージェント向けに `memory` フロントマターフィールドのサポートを追加し、`user`、`project`、`local` スコープでの永続メモリを有効化 (v2.1.33)

### 新タスク管理システム
- 依存関係追跡などの新機能を含む新しいタスク管理システムを追加 (v2.1.16)
- `TaskUpdate` ツールによるタスクの削除機能を追加 (v2.1.20)

### キーバインドカスタマイズ
- カスタマイズ可能なキーボードショートカットを追加 (v2.1.18)
  - コンテキストごとにキーバインドを設定し、コードシーケンスを作成し、ワークフローをパーソナライズ可能
  - `/keybindings` を実行して開始
- 設定可能な複数行入力のための `chat:newline` キーバインドアクションを追加 (v2.1.47)

### バックグラウンドエージェント強化
- `background: true` を指定して常にバックグラウンドタスクとして実行するサポートを追加 (v2.1.49)
- ESC の二重押しの代わりに Ctrl+F ですべてのバックグラウンドエージェントを終了するように変更 (v2.1.47)
- バックグラウンドエージェントが起動前にツール権限の確認を求めるように変更 (v2.1.20)

### ConfigChange フック
- セッション中に設定ファイルが変更された際に発火する `ConfigChange` フックイベントを追加 (v2.1.49)
  - エンタープライズセキュリティ監査や設定変更のオプションブロックが可能に

### PR 連携強化
- 特定の GitHub PR 番号または URL にリンクされたセッションを再開する `--from-pr` フラグを追加 (v2.1.27)
- `gh pr create` で作成された際にセッションが自動的に PR にリンクされるように (v2.1.27)
- プロンプトフッターに PR レビューステータスインジケーターを追加 (v2.1.20)

### その他の注目機能
- スラッシュコマンドとスキルを統合し、メンタルモデルを簡素化 (v2.1.3)
- 到達不可能な権限ルールの検出と警告を追加 (v2.1.3)
- 「ここから要約」をメッセージセレクターに追加し、会話の部分的な要約が可能に (v2.1.32)
- 現在のセッションのトラブルシューティング用に `/debug` を追加 (v2.1.30)

## 2. モデル・速度アップグレード

### Claude Opus 4.6
- Claude Opus 4.6 が利用可能に (v2.1.32)
- Opus 4.6 エフォートの一回限りの案内を対象ユーザーに追加 (v2.1.42)

### Claude Sonnet 4.6
- Claude Sonnet 4.6 のサポートを追加 (v2.1.45)
- Max プランから 1M コンテキストの Sonnet 4.5 が削除され、最新の Sonnet 4.6 に移行 (v2.1.49)

### Fast Mode
- Opus 4.6 でファストモードが利用可能に (v2.1.36)
  - 詳細: https://code.claude.com/docs/en/fast-mode

## 3. パフォーマンス改善

### 起動高速化（複数バージョンで継続的に改善）
- Zod スキーマの構築を遅延させることで起動パフォーマンス改善 (v2.1.42)
- SessionStart フックの実行を遅延させ、インタラクティブまでの時間を約500ms短縮 (v2.1.47)
- MCP 認証失敗のキャッシュにより冗長な接続試行を回避 (v2.1.49)
- アナリティクストークンカウントの HTTP 呼び出しを削減 (v2.1.49)
- MCP ツールのトークンカウントを単一の API 呼び出しにバッチ化 (v2.1.49)
- 非インタラクティブモード（`-p`）の不要な API 呼び出しをスキップ (v2.1.49)
- 統計キャッシュ用のセッション履歴の即時読み込みを削除 (v2.1.45)
- React Compiler による UI レンダリングパフォーマンス改善 (v2.1.15)

### メモリ使用量改善
- 進行状況更新での O(n^2) メッセージ蓄積を排除 (v2.1.47)
- API ストリームバッファ、エージェントコンテキスト、スキル状態を使用後に解放 (v2.1.47)
- エージェントタスク完了後にメッセージ履歴をトリミング (v2.1.47)
- 大量の出力を生成するシェルコマンドの RSS が無制限に増加しなくなった (v2.1.45)
- `--resume` のメモリ使用量を改善（多数のセッションを持つユーザーで 68% 削減） (v2.1.30)

### メモリリーク修正
- tree-sitter のパースツリー未解放による WASM メモリの際限ない増加を修正 (v2.1.2)
- Yoga WASM のリニアメモリが縮小しないことに起因するメモリの無制限な増加を修正 (v2.1.49)
- シェルコマンド完了後のストリームリソース未クリーンアップによるメモリリークを修正 (v2.1.14)
- git diff パースでスライスされた文字列が大きな親文字列を保持してしまうメモリリークを修正 (v2.1.0/関連修正)

### ファイル候補・検索高速化
- `@` ファイルメンションのパフォーマンスを改善 - 起動時にインデックスをプレウォームし、バックグラウンドリフレッシュ付きのセッションベースキャッシュを使用 (v2.1.47)

## 4. セキュリティ修正

### コマンドインジェクション・権限バイパス
- bash コマンド処理のコマンドインジェクション脆弱性を修正 (v2.1.2)
  - 不正な入力により任意のコマンドが実行される可能性があった
- シェルの行継続を利用した権限バイパスを修正 (v2.1.6)
  - ブロックされたコマンドが実行される可能性があった
- ワイルドカード権限ルールがシェル演算子を含む複合コマンドにマッチできる脆弱性を修正 (v2.1.7)
- サンドボックスから除外されたコマンドが `autoAllowBashIfSandboxed` が有効な場合に Bash ask 権限をバイパスできるバグを修正 (v2.1.34)
- コマンドスマグリングを防止するための heredoc デリミタ解析を改善 (v2.1.38)

### ハルシネーション防止
- bash 権限分類器で、返されたマッチ説明が実際の入力ルールに対応しているかを検証 (v2.1.47)
  - ハルシネーションによる説明が誤って権限を付与することを防止

### データ露出防止
- 分析イベントに MCP ツール名が露出する問題を修正 (v2.1.2)
  - ユーザー固有のサーバー設定をサニタイズ

## 5. Breaking Changes・非推奨

### npm インストール非推奨
- npm インストールの非推奨通知を追加 (v2.1.15)
  - `claude install` を実行するか公式ドキュメントで他のオプションを確認

### Sonnet 4.5 (1M コンテキスト) → Sonnet 4.6 移行
- Max プランから 1M コンテキストの Sonnet 4.5 が削除 (v2.1.49)
  - 1M コンテキストを持つ最新の Sonnet 4.6 に移行
  - `/model` で切り替え

### 設定ファイル・パスの変更
- 設定バックアップファイルをホームディレクトリルートから `~/.claude/backups/` に移動 (v2.1.47)
- Windows のマネージド設定パス `C:\ProgramData\ClaudeCode\managed-settings.json` を非推奨化 (v2.1.2)
  - `C:\Program Files\ClaudeCode\managed-settings.json` に移行が必要

### ツールフックタイムアウト変更
- ツールフック実行のタイムアウトを60秒から10分に変更 (v2.1.3)

## 6. 日常で便利な改善

### 日本語・CJK サポート
- 日本語 IME からの全角スペース入力のチェックボックス選択でのサポートを追加 (v2.1.31)
- 日本語 IME からのオプション選択プロンプトでの全角数字入力のサポートを追加 (v2.1.21)
- CJK 全角文字がタイムスタンプやレイアウト要素のずれを引き起こす問題を修正 (v2.1.47)

### PDF ページ指定読み込み
- Read ツールに PDF 用の `pages` パラメータを追加し、特定のページ範囲の読み取りが可能に (v2.1.30)
  - 例: `pages: "1-5"`
  - 大きな PDF（10ページ超）は `@` メンション時にコンテキストにインライン展開されず、軽量な参照を返すように

### スピナーカスタマイズ
- カスタマイズ可能なスピナー動詞設定（`spinnerVerbs`）を追加 (v2.1.23)
- スピナーチップをカスタマイズするための `spinnerTipsOverride` 設定を追加 (v2.1.45)

### セッション管理の改善
- カスタムセッションタイトルが会話再開後に失われる問題を修正 (v2.1.47)
- コンテキストコンパクション後にセッション名が失われる問題を修正 (v2.1.47)
- resume ピッカーの初期セッション数を10から50に増加 (v2.1.47)

### MCP ツール検索の自動化
- すべてのユーザーに対して MCP ツール検索の自動モードをデフォルトで有効化 (v2.1.7)
  - MCP ツールの説明がコンテキストウィンドウの10%を超える場合、自動的に遅延検出
  - 多数の MCP ツールを設定しているユーザーのコンテキスト使用量が削減

### auth 関連
- `claude auth login`、`claude auth status`、`claude auth logout` CLI サブコマンドを追加 (v2.1.41)

### bash 履歴補完
- bash モード（`!`）での履歴ベースのオートコンプリートを追加 (v2.1.14)
  - コマンドの一部を入力して Tab を押すと bash コマンド履歴から補完

### プラグイン固定
- プラグインを特定の git コミット SHA に固定するサポートを追加 (v2.1.14)

### コンテキストウィンドウ表示
- ステータスライン入力に `context_window.used_percentage` と `context_window.remaining_percentage` フィールドを追加 (v2.1.6)

### VSCode 連携改善
- VS Code のプランプレビューを改善: Claude の反復に応じて自動更新 (v2.1.47)
- VSCode: リモートセッションのサポートを追加し、OAuth ユーザーが claude.ai からセッションを閲覧・再開可能に (v2.1.33)
- VSCode: セッションピッカーに git ブランチとメッセージ数を追加 (v2.1.33)
