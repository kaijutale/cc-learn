- 実装内容:
  - **`/compact` と `/clear` の違い整理**:
    - `/clear` = 履歴完全リセット（新タスク切替時に使用）
    - `/compact` = 会話要約で圧縮（同タスク継続で息切れ時に使用、オプションで要約焦点の指定可 例: `/compact テスト結果だけ残して`）
    - `autocompact` がコンテキスト上限接近時に裏で自動発動する点も確認
    - 使い分けの勘所: **タスク切替→`/clear`**、**同タスク継続で息切れ→`/compact`**
  - **コンテキストエンジニアリングの一行定義確立**:
    - 「LLMのワーキングメモリ（= コンテキストウィンドウ）に、いつ・何を・どんな形で載せるかを設計する実践」
    - プロンプトエンジニアリング（= 料理のレシピ）の1階層上（= 厨房の動線・仕込み・道具配置）
    - Andrej Karpathy / Tobi Lütke らが2024年頃から強調し始めた概念、"LLM時代のシステム設計論"の位置づけ
  - **コンテキストに載る要素の棚卸し**:
    - System Prompt / Skills / Tool定義 / Knowledge（RAG, ファイル, 記憶） / 会話履歴 / Input Format / Output Format の7層
  - **静的 × 動的の2軸フレームワーク化** ← セッションの最大の収穫:
    - **静的設計**（作業スペース構築）: CLAUDE.md / Skills定義 / フック / 役割ファイル / references  ← 1回の投資
    - **動的運用**（作業スペース管理）: `/compact`, `/clear`, ToolSearch, サブエージェント分離, 記憶の書き込み/読み出し  ← 毎ターンの判断
    - 比喩統合: 「厨房のレイアウト」（静的）と「仕込み・動線・次の料理判断」（動的）の両輪
  - **記事引用との統合解釈**: 
    - 記事が「ロングコンテキストは主役ではなく体力」「主役は関心事の分離と反復可能なチーム運用」と言う根拠 = **動的運用こそが差を生む部分**だから

- 設計意図:
  - **気づきログとして記録した理由**: かもねの「仕組みの深い理解を重視する学習スタイル」（MEMORY.md user_claude-code-learner.md）に沿って、一度構造化した概念を再利用可能な形で永続化。`/clear` で消えない場所に置くことで、次回以降のセッションで概念参照が可能になる
  - **静的×動的の2軸化を選んだ理由**: 単に「環境設計」と一言で済ませると、動的運用の重要性（`/compact` 判断、ToolSearchの必要性、定期的な重要情報の再注入等）が見えなくなる。2軸に分離することで「毎ターン何を判断するか」が可視化される。かもねの要約「環境（作業スペース）の設計・構築」に対し、静的側面は正しく捉えているが動的側面が欠けている点を補完した
  - **Lost in the Middle 問題への接続**: 研究で実証された「長い文脈の中盤は読み飛ばされやすい」現象を、動的運用の必要性の**実証的根拠**として採用。体感や比喩だけでなく、客観的な理由付けを与えることで説得力を強化
  - **既存ログ（5-Role Separation Framework）との関係設計**: 前回ログは記事引用を**実装**に変換した成果物、本ログは記事引用を**概念**に変換した成果物。実装 / 概念の両輪として対をなす位置づけ
- 副作用:
  - **実装ログ形式との形式的不一致**: 本スキルは「実装行為の記録」を想定した設計。気づきログとして運用すると、"設計意図" / "副作用" 欄の意味合いがずれる（実装の意図ではなく「記録の意図」になっている）。長期的には `.docs/knowledge/` や `.docs/notes/` など別カテゴリへの分離が望ましい。本ログは"実装ログではない"というタグを機能名に含めることで暫定的に区別した
  - **"実装内容"欄の歪み**: 本来はコード/ファイル変更の記録欄。今回は概念整理の内容を入れたため、将来 grep 時に「実装したもの」を検索した際にノイズになる可能性。対策として機能名に "気づきログ" を明記
  - **ファイル命名の機能名感の揺らぎ**: `context-engineering-understanding.md` は具体的成果物名ではなく概念名。既存の `5-role-separation-framework.md`（具体的成果物名）と並べると識別性の軸がずれる
