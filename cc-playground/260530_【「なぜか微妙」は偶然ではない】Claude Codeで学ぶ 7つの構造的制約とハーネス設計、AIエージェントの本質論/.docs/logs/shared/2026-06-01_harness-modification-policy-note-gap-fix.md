# harness-modification-policy が note の本質から逸れている件の調査と P0 修正

> 9体の subagent で note 9本を独立コンテキスト精読し 2rule と照合。「新規追加がデフォルト」が note の本質から逸脱と判定、20 問題を抽出し P0(L2 逆転)を修正。

## 概要

検証対象は 2 つの rule:
1. `~/.claude/CLAUDE.md` の 1 行「協調ハーネス改修: 契約(依存先)破壊は不可、新規追加がデフォルト (後方互換な改善・バグ修正は可)」
2. `~/.claude/rules/harness-modification-policy.md` 本体

問い: これらは note-articles(まさお氏)9本の本質を捉えているか。捉えていないなら何が逸れているか。

## 内容

### 調査方法

- 9 体の subagent(general-purpose)を独立コンテキストで並列起動。1 体 = 1 note。
- note 9 本(260222 / 260304 / 260308 / 260321 / 260404 / 260405 / 260418 / 260426 / 260517)を各全ページ精読。
- 統一スキーマで 2rule と逐条照合。反迎合指示(SUPPORT / CONTRADICT / OUT-OF-SCOPE を区別、gap を能動探索、問題ゼロを疑う)。

### 判定分布

- 中立(射程外): 8 本 / 部分的に支持: 1 本(260517 Skill設計)/ 強く支持: 0 / 反証: 0
- **全 9 本に通底**: 「凍結ではない = 薄く作って反復的に深める」← rule が唯一正しく捉えた本質
- 「契約」語彙の出典: 260517(p8「責務を契約として閉じる」)/「3層」の出典: 260222(p30-32 壁/地図/脳)
- 「新規追加がデフォルト」: **出典 note 無し**(BIBLIOGRAPHY.md にも未登録)+ 複数 note と緊張

### 核心: 「新規追加がデフォルト」が note の本質から逸脱

- note の幹: 「既存を参照・改修して深める(260222 L2 = 地図 = 参照)」+「肥大化したら削れ(260426/260517)」
- rule は「連鎖崩壊防止(①)」だけ最適化し「肥大化・腐敗防止(②)」を捨てた**片側最適化**
- **震源**: policy.md の L2 が、260222 の「地図 = 参照パターン」から「新規作成デフォルト」にすり替わり、メタファが破綻(地図 = 既存を参照、なのに中身が新規作成)。「8 割」という数字だけ共通で意味が反転(note = 参照が 8 割カバー / rule = 新規が 8 割正しい)

### P0 修正(実行/提示)

- **①policy.md L2(反映済み)**: 「新規デフォルト」→「既存を参照し契約を壊さない後方互換改修で深めるのがデフォルト。新規作成は契約を壊す改修しか手が無い時の退避策」。260222 の L2=地図=参照を復元。
- **②CLAUDE.md 127行サマリ(提示・かもね手動)**: 重心を「既存改修で深める」に移動。before「契約破壊は不可、新規追加がデフォルト」→ after「既存を参照し契約を壊さず後方互換に改修して深めるのがデフォルト。契約を壊す時だけ新規作成」。
- **効果**: 問題 1,3,5,18 が連鎖的に解消。L2 が判断軸節(5-12行「依存先を壊さない→改修してよい」)と整合した。

## 設計意図

- P0 スコープを「重心修正」一点に厳密化。前回案でサマリに入れていた「肥大化したら棚卸し・削除」は外した — 宣言するなら本体に棚卸し節(P1 改修③)が必要で、サマリ先走りは本体と不整合になるため。
- 方針は「①連鎖崩壊防止を捨てず退避策として残し、②肥大化防止をデフォルトに昇格」=「①を残し②を足す」。①の動機(連鎖崩壊防止)は正当なので消さない。
