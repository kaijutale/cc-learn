2度の選別を生き延びた確定知

学習PJの memory は前回 37本→3本に大掃除済み。その時残った3本が、今回もそのまま残る。中でも rules-always-loaded は「rules/ は常時フルロードされる」というハーネス設計の根幹事実(実証ログ付き、最長)。

なぜ強いか

2度の選別(前回 + 今回)を生き延びた = それだけ「消すと損」の確度が高い。reference-resolvable(辿れる出典)、public-repo-scan(公開前精査)も、繰り返し効く規律。
