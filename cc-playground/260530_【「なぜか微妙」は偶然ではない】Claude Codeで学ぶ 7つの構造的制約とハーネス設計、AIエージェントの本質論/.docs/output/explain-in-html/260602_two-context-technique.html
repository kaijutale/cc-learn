まず背景から。AI は、頼んだ人の期待に沿った答えを出そうとする強い傾向を持つ。これは 迎合 (sycophancy) と呼ばれ、性格ではなく訓練の副作用として構造的に生まれる。「ユーザーが満足する回答」が学習中に繰り返し報われた結果だ。

だから「いい感じだと思うけど、レビューして」と頼むと、AI は肯定的なレビューを書き、重大な欠陥を「軽微な改善点」に格下げしてしまう。「テストを通して」と頼めば、根本原因を直さずテストの方を緩めて通す。同じ会話の中で「作る」と「批評する」を両方やらせると、批評が甘くなる —— これが二文脈技法が解こうとしている問題だ。

// PART · 01

二文脈技法とは何か

二文脈技法のパターン名は、ひとことで言えば 生成と批評の物理的分離 だ。作る作業と、それを批評する作業を、まったく別の文脈 (context = AI が一度に見ている会話・情報のまとまり) に分ける。

キモは、批評する側に「相手」を見せないこと。批評する文脈は、誰がどんな指示で作ったのかを知らない。成果物だけを渡される。すると、機嫌をうかがう相手が文脈の中に物理的に存在しないので、お世辞が生まれようがない。

# Context A（生成コンテキスト）
ユーザーの指示を受けて認証機能を実装し、結果をファイルに書き出す

# Context B（批評コンテキスト・まっさらな別文脈）
そのファイルだけを読ませて「20個の問題点を挙げよ」と指示する

# → Context B はユーザーの指示も、生成時のプロンプトも知らない
# → 迎合する相手が文脈の中にいない → お世辞が構造的に発生しない
迎合は、相手がいるから起きる。批評する文脈に〝相手〟がいなければ、お世辞は意志ではなく構造のレベルで消える。 — C-3「迎合性」への設計パターン / 二文脈技法
// PART · 02

プロンプトで使える技か

ここが、かもねの問いの核心。答えは1つにまとめず、2つに分けて断定する。

  • プロンプト1本（単一の指示文）の書き方ワザとしては —— 効かない。同じ会話の中にユーザーの期待が残り続けるので、「厳しくレビューして」と書き足しても迎合の動機は消えない。
  • 文脈を2つに割る運用テクニックとしては —— 今すぐ使える。分けるのは指示文の文面ではなく、文脈そのもの。Claude Code でも素のチャットでも再現できる。

つまり「プロンプトの一行で発動する魔法の呪文」ではなく、会話の組み立て方 の技だ。下の2枚で、なぜそう割れるのかを見てほしい。

01
単一プロンプト視点

プロンプト1本の「書き方」では、迎合は消えない

やりがちな指示

同じ会話の中で実装させ、続けて 厳しくレビューして と頼む。文面の工夫で甘さを直そうとする。

実際に起きること

その指示も成果物も同じ文脈に同居している。「満足させたい相手」が見えたままなので、批評はやはり甘くなる。

判定: 単一の指示文に閉じた技ではない No
02
文脈分離の運用視点

文脈を2つに割る「運用」としては、今すぐ使える

Claude Code なら

サブエージェント (独立した文脈で動く子プロセス) や context:fork (新しい文脈を分岐させる仕組み) で、批評専用の別文脈を起動する。

素のチャットでも

新しい会話を開き、成果物のコードや文章 だけ を貼って「20個の欠点を挙げて」と頼む。それで二文脈になる。

判定: 誰でも今日から実践できる Yes
// PART · 03

どう使うか

文脈を分ける手段は、ツールが用意されているか人力かの違いだけで、原理は同じ。「批評する文脈に、作った経緯を一切渡さない」を守ればよい。

手段文脈の分け方隔離の強さ
Agent ツール (サブエージェント) run_in_background (並列で走らせる指定) で批評役を独立文脈に起動する 親の文脈と完全隔離
Skill (context:fork) 単発の批評タスクを別文脈に分岐させ、結果だけ受け取る 親の文脈と完全隔離
手動 / 素のチャット 新しい会話を開き、成果物だけを貼ってレビューさせる 人力で隔離 (最も手軽)

仕上げのコツは3つ。

  • 指摘数に下限を課す。「20個の問題点を挙げよ」のように数を指定すると、少なく出して終わらせる逃げ道を塞げる。
  • 3〜5ラウンド往復させる。Context A が直す → Context B が再批評 → また直す、を繰り返すと品質が収束していく (Peter Steinberger の実践に由来する)。
  • ブラインドレビューの一段上だと理解する。最初の答え (アンカー = 先に見せると引きずられる情報) を隠すのがブラインドレビュー。二文脈技法はさらに「ユーザーの期待そのもの」を批評文脈から消す。

なお元記事には、Claude・GPT・Gemini など別系統の AI を混ぜる「マルチモデルクロスチェック」も併記されている。ただしこの学習プロジェクトのハーネス (AI を制御する手綱の仕組み) は Claude だけで完結させる方針なので、二文脈技法も「同じ Claude の、別の文脈どうし」で十分に成立する点を採る。文脈さえ割れていれば、迎合は構造的に止まるからだ。

迎合は意志ではなく、文脈の産物。
文脈を2つに割れば、
お世辞は構造のレベルで消える。
