03横断する 5 つの構造的根因

60 件は別物に見えて、根は 5 つの設計上の癖に収束する。修正は対症療法でなくこの 5 つを断つのが本筋じゃ。

  • 1. 人間可読の文章を正規表現で読む — 禁止語の部分一致、CLAUDE.md の誤報告、設定行の取りこぼし、essence-gate の散文解析。書式が少し変わるだけで誤抽出・抽出失敗する。
  • 2. 失敗時に素通りさせる (fail-open) — 絵文字の grep 不発、空フェーズファイル、秘密ファイルのファイル名のみ、応答終了検査の見逃し。検査不能を「合格」と取り違える。
  • 3. 特定の命名・ツール名を決め打つ — 編集ツール MultiEdit 未対応、essence-gate のファイル名 glob、通知のサーバー名解析。命名が変わると追従できない。
  • 4. 境界の数え方を誤る — 行数カウントの 1 行ずれ、区切りの閉じ忘れ見逃し、改行コードや先頭の見えない文字での誤判定。
  • 5. シェル変数を素で展開する — コマンド検査の語分断、禁止語ループ、書込先のワイルドカード誤展開。引用符を付けないことで意図せぬ分割・展開が起きる。
