03外部化の利得 4 点は実運用されているか

  • バージョン管理 — ハーネス (~/.claude) 自体が git リポジトリで、GitHub (claude-harness) に遠隔バックアップされてる。方針の全変更履歴が残る。
  • diff 一発の方針比較 — 方針変更は commit / PR の差分としてレビューされる。「設定ファイルだから diff できる」を毎日運用中。
  • 監査 — 07-11〜07-19 の観測レビューシリーズは、まさに方針 diff の監査をセッション横断で回した実績 (変更履歴を 7 つの制約で批評し issue 化)。
  • 移植 — establishing-knowledge-persistence 等の skill が、方針一式を他プロジェクトへ複製 (scaffold) する仕組みを持つ。
