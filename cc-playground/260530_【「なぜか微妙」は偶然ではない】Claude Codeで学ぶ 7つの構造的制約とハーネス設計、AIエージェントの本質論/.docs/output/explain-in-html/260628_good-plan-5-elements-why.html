01結論 — 5要素を一言ずつ

良い計画とは、やることを並べたTODO リストではなく、それ単体で再開・実行・確認できる「引き継ぎ書」じゃ。そのために欠かせぬのが次の 5 つ。まずは一言で掴め。

  • 1. 目的 — 何を達成すれば完了か。
  • 2. 前提 — 何が必要か (触るファイル・環境・依存・参照する実装)。
  • 3. 判断理由 — なぜこの方法を選んだか (捨てた代替案も)。
  • 4. 検証 — どう確かめるか (人の主観でなく、機械が合否を出せる形で)。
  • 5. 進捗 — どこまで終わったか (チェック + commit の印 + 今いる場所)。

02そもそも、なぜ「計画」が要るのか

AI (大規模言語モデル) には決定的な性質がある — ステートレス (記憶を持たず、セッションが終わると前回の文脈がまるごとゼロに戻る) ということ。

たとえるなら — 腕は一流だが、部屋を出るたびに記憶を全部失う同僚。次に呼ぶときは「何も知らない新人」として現れる。その相手に「あとはよろしく」とだけ言って渡せるか? 渡せぬ。だから計画は、その同僚が一人で読んで、一人で再開できる「引き継ぎ書」でなければならぬ。

5 つの要素は、この引き継ぎ書が絶対に落としてはいけない問いを 1 つずつ埋めるためのもの。裏付けとして記事はこう引く — Steinberger 「計画ファイルを再起動可能な状態 (途中から再開できる形) として設計せよ」、Ronacher 「5 ステップを超える作業なら、行き当たりばったりでなく計画フェーズを置け」。

035要素を1つずつ — なにを答え、無いと何が起きるか

各要素は「答える問い」と「それが無いときに必ず起きる事故」が一対一で対応しておる。事故 (赤字) を見れば、なぜ必須かが腑に落ちるはずじゃ。

1. 目的何を達成するか

これは何このタスクが「完了」と言える到達点。ゴールそのもの。
無いとドリフト (目標からのズレ) が起きる。手段をいじるうちに「そもそも何のためだったか」を見失い、手段が目的にすり替わる。
たとえ行き先を決めずに車を出すようなもの。走ってはいるが、着いたかどうかを誰も判定できぬ。

2. 前提何が必要か

これは何作業に必要な材料 — 触るファイル、動かす環境、依存ライブラリ、お手本にする参照実装。
無いと存在しないファイルを参照したり、環境の違いで失敗する。記憶ゼロの相手は「どこに何があるか」を自力で知らぬ。
たとえ料理のレシピで材料表を省くようなもの。手順だけ渡されても、何を冷蔵庫から出すかが分からぬ。

3. 判断理由なぜこの方法か

これは何その方法を選んだ理由。そして検討した上で却下した代替案も含める。
無いと次のセッションが別のやり方で作り直し、前の判断と矛盾する。「なぜ A でなく B にしたか」が残っていないと、同じ議論を何度も蒸し返す。
たとえ「あえてこの道を選んだ」というメモが無いと、後から来た人が「近道があるのに」と勝手に作り変えて、地雷を踏み直す。

4. 検証どう確認するか

これは何「できた」を確かめる手段。各ステップを機械が合否を出せる形 (テストが通る / exit 0 で終わる等) で書く。
無いとAI の「できました」という自己申告を検出できない。本当は途中なのに完了と宣言される「早期完了宣言」を見抜けぬ。
たとえテストの答案を自分で採点させるようなもの。「満点です」と言われても、客観的な採点基準が無ければ信じる根拠が無い。

5. 進捗どこまで終わったか

これは何現在地の記録 — チェックボックス + commit の印 (どの変更まで保存したか) + 今いる場所。
無いと同じ作業を二重にやり直したり、逆にやり残しを見落とす。記憶ゼロの相手は「どこまで進めたか」を毎回ゼロから推測するしかない。
たとえ登山の途中に道標が無いようなもの。同じ斜面を登り直すか、まだ先があるのに頂上だと勘違いする。

04ただの TODO リストとの違い

「やることリスト」と「良い計画」の差は、この 5 要素が入っているかどうかに尽きる。最後の行 —「一人で再開できるか」— が両者を分ける決定打じゃ。

観点ただの TODO リスト良い計画 (5要素入り)
目的欠「○○を直す」だけ有何を満たせば完了かまで書く
前提欠書かない有触るファイル・環境・参照実装を明記
判断理由欠書かない有なぜこの方法か・却下案も残す
検証主観「動けば OK」有機械が合否を出せる形で書く
進捗浅チェックのみ有チェック + commit の印 + 現在地
一人で再開できる?不可頭の中の文脈が要る可単体で再開・実行・確認できる

TODO リストは「書いた本人の記憶」に寄りかかっておる。良い計画は記憶に頼らぬ — だからこそ、記憶を持たない AI にも、明日の自分にも、別の誰かにも、そのまま手渡せる。

05総評

良い計画の 5 要素 — 目的・前提・判断理由・検証・進捗 — は、AI が記憶を持たない (ステートレスな) 存在だという前提から逆算して生まれた。新しいセッションは毎回「何も知らない新人」として現れるから、計画は本人の頭の中に頼らず、一人で読んで再開できる引き継ぎ書でなければならぬ。5 つはそれぞれ「何を達成するか / 何が必要か / なぜこの方法か / どう確かめるか / どこまで進んだか」という、引き継ぎ書が絶対に落としてはいけない問いを 1 つずつ埋める。逆に言えば、この 5 つのどれかが欠けた瞬間に、ドリフト・参照ミス・作り直し・偽の完了・二重作業という、決まった事故が顔を出す。だから 5 つは飾りでなく、必須なのじゃ。
