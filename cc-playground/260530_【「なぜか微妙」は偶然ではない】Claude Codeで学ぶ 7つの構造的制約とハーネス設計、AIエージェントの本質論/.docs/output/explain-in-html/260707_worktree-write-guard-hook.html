02Before ― 警備員がいない世界の事故

事故複製店にいたのに、本店の壁を壊した

何が起きたか改装業者(worktree のセッション)が、習慣で本店の住所を思い出し、営業中の本店 ~/.claude/skills/review-agent-essence/SKILL.md を直接 Edit してしまった。
正体複製店の中で作業していたはずが、編集先を「今いる場所(複製店)」でなく「記憶の絶対パス(本店)」で決め打ちした。これが issue #76 で実際に起きた漏れじゃ。
怖い点本店が git で追えぬまま汚れ、最悪 気付かず PR まで進む。「本体が最新」という誤魔化しに化ける。

03看板(CLAUDE_CONFIG_DIR)は「壁」ではない

半分だけ道案内はするが、入店は止められぬ

看板の役目CLAUDE_CONFIG_DIR="$PWD" は「あなたの現場は複製店です」と示す看板。セッションが読むハーネスを複製店の中身に切り替え、作業メモ(current-issue.md)も読ませる。
看板の限界しかし本店の扉に鍵はかからぬ。業者が看板を無視して"本店の住所"へ歩けば、そのまま入って壁を壊せる。＝ 絶対パスへの書き込みは看板では止まらない。

04After ― #97 は入口に警備員を立てる

構造で根絶複製店担当の本店入りを、門前払い(deny)

やること本店の入口に警備員(PreToolUse フック)を常駐させる。複製店担当のセッションが本店 ~/.claude/ を編集しようとしたその瞬間、実行前に拒否(deny)して止める。
効果事故は「起きたら直す」から「そもそも起こせない」へ変わる。業者の注意力・善意に頼らず、構造で根絶する。

05警備員は何を見て止めるか(技術面)

フックは、編集ツールが走る直前に次の3条件を全て満たす時だけ止める:

もし  : 複製店として起動している (CLAUDE_CONFIG_DIR が設定済 かつ ≠ 本体 ~/.claude)
 かつ : 編集先が 本店 ~/.claude/ の中
 かつ : それが 複製店(CLAUDE_CONFIG_DIR)の外
ならば: → 実行前に拒否 (deny)。理由を添えて止める

本店で起動した通常作業(看板なし＝CLAUDE_CONFIG_DIR 未設定)では発火しない。ゆえ普段のハーネス改修は一切妨げぬ。挙動を並べるとこう:

状況ガード無し(以前)#97 あり
複製店から本店の skill を Edit漏れるblock
複製店の中の skill を Edit通る通る
一時置き場(/tmp)へ Write通る通る
本店起動で本店を Edit(通常作業)通る通る

06正直に ― まだ塞がらぬ抜け道

#97 の第1弾は「編集ツール(Edit/Write 系)」を塞ぐ。＝ 実際に事故った道はこれで根絶。ただし鉄壁ではない:

  • 裏口 ― Bash での書き込み — echo > ~/.claude/... や git -C ~/.claude のような Bash 経由の書き込みは、編集ツールでないため第1弾では素通り。第2弾(任意)で主要パターンだけ塞ぐ予定。
  • 看板の付け忘れ — 効くのは CLAUDE_CONFIG_DIR を付けて起動した時だけ。素の claude(本店起動扱い)では警備員が立たぬ。

07総評

ガード無しの安全は「業者が住所を間違えない」という注意力頼み ― 人は必ず間違える。#97 は入口に警備員を置き、間違えても入れない構造にする。実際に事故った編集ツールの経路を決定論で塞ぐ、これが本質の一手じゃ。
