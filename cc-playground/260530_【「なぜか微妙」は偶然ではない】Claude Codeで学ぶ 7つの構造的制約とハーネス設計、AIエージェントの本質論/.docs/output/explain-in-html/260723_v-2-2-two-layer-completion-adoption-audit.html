03記事超え点

  • 強制度の勾配 — note は構造チェックを一律「CI や Hook で自動実行」と説くが、ハーネスはチェックの決定可能性で強制度を変える: 配置準拠 (パスの正誤=決定論) は block+再指示、行数 (分割すべきかは凝集度=意味論判断) は warn。決定論/意味論境界を構造チェック内部にも適用。
  • structure 概念のメタ層拡張 — コードのメトリクスに加え、設計原則層 (essence verdict が skill/agent/hook の commit を gate = ハーネス自身の保守可能性) と文書グラフ層 (ticket 循環検出 = 知識ベースの構造健全性) へ二層判定を延長。
