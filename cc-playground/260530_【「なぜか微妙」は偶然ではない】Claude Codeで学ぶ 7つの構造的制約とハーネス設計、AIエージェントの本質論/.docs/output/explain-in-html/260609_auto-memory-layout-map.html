まず大原則 ── auto-memory は プロジェクトごとに隔離されておる。保存場所は必ず:

  • ~/.claude/projects/ の下に、プロジェクト1つにつき1ディレクトリ
  • そのディレクトリ名(スラッグ)は、元の絶対パスの / を - に置換したもの。例: /Users/camone/.claude → -Users-camone--claude
  • 各ディレクトリの中に memory/ があり、その中に MEMORY.md(索引)と本文 .md が並ぶ

// PATTERNS · 02

配置の3つの型

A
索引型 · 4PJ・10本

MEMORY.md(索引) + 個別 .md(本文)

標準の形。MEMORY.md が - [タイトル](ファイル名.md) — 要約 の行で各本文を**索引**し、本文は個別 .md として並ぶ。~/.claude(4)・learn(3)・ios-memopo(2)・ios-ideas(1)がこれ。MEMORY.md の先頭が常時ロードされるので、索引は軽く保つ。

B
直書き型 · 2PJ・2本

MEMORY.md に本文を直接書く

toybox と ninmu は本文ファイルを持たず、MEMORY.md 自体に技術スタックや gotcha を**直書き**しておる。本文1ファイルで完結する小規模・休止中PJの形。索引と本文が一体化した姿じゃ。

C
スラッグの読み方

ディレクトリ名から元PJを逆算

ディレクトリ名は元の絶対パスの / を - に変えたもの:

  • -Users-camone--claude → /Users/camone/.claude(/.claude が --claude と二重ハイフンになる)
  • -Users-camone-dev-projects-toybox → /Users/camone/dev/projects/toybox

逆に辿れば、memory が**どのプロジェクトの記憶か**が一目で分かる。

記憶はプロジェクトごとに隔離され、
スラッグがその出身地を指し示す。
12項目それぞれに、確かな住所がある。 — 整理後の配置地図より

どこに何が住むかが分かれば、
記憶は迷子にならぬ。
6つの棚に、12の記憶が整然と並ぶ。
