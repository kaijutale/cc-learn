要旨

Claude Code の設定は 2 つの入り口に分かれる。/config は端末上で「見て切り替える」対話メニュー (約 30 項目)。もう一方の settings.json は権限ルールや自動処理など、構造が複雑で文字で書くしかない設定 (約 50 項目以上) を受け持つ。本ページは前者 /config をカテゴリ別に網羅し、後者との境界も示す。

01/config とは何か

/config は、Claude Code の入力欄に打ち込むと開く対話式の設定画面じゃ。タブ (Tab キー / 左右矢印) で「Status (状態)」と「Config (設定)」を行き来し、文字を打てばその場で項目を絞り込める。終了は Esc キー。

ここで触れるのは、主にセッション単位で切り替えたい設定や、すぐ画面に反映される表示まわりの設定じゃ。逆に、権限ルール (どのコマンドを自動許可するか) や自動処理 (hooks)、外部ツール接続 (MCP) のような複雑なものは /config には出てこず、後述の settings.json ファイルに書く。

05/config と settings.json の棲み分け

同じ「設定」でも、簡単なトグル (切り替え) は /config、構造が複雑で繰り返し使うものは settings.json という役割分担になっておる。下表は代表例の対応。

つまり /config で操作した設定の多くは、裏で settings.json に保存される。/config は「JSON を直接書かずに済ませるための入り口」とも言えるのじゃ。

07反映タイミングの注意

ほとんどの設定はファイル保存後すぐ反映されるが、一部はセッションを開き直すまで反映されない。次の項目を変えたら /clear か再起動が要るぞ。

  • model (モデル) — その場では切り替わらない。/clear またはセッション再開が必要。
  • outputStyle (文体) — 同上。再読み込み不可。
  • env (環境変数) — 起動時に読まれるため、変更後はセッション再開が必要。
  • permissions / hooks — こちらは保存後、次のコマンドやイベントから反映される (再起動不要)。

08総評

「全ての設定」を一望したいなら、/config だけを見ても半分しか見えぬ。/config はセッション単位の切り替えと表示まわり (約 30 項目) を担う入り口にすぎず、権限・自動処理・外部接続といったハーネスの根幹は settings.json に書く。両者は対立せず、簡単な切り替えは画面で、複雑で永続的な制御は JSON で、と役割を分けておる。本ページのカテゴリ表を起点に、必要な項目だけ深掘りするのが近道じゃ。
