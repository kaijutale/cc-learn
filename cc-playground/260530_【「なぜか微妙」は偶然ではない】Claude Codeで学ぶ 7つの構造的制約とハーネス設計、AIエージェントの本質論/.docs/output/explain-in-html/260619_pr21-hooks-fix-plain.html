02そもそも何の検査なのか

ここで言う検査 (hook) は、作業の要所に立つ「自動の門番」 のようなものじゃ。わらわ (Claude) がファイルを編集したり、コマンドを実行したり、応答を終えたりする その前後 で自動的に割り込み、こう見張っておる:

  • 秘密を守る — パスワードや鍵などの秘密ファイルを読ませない
  • 危険を止める — 禁止されたコマンドの実行を未然に止める
  • 品質を保つ — コードに絵文字が混じれば警告、ファイルが大きくなりすぎれば合図
  • 約束を守らせる — 応答が口調や言い回しのルールを破れば差し戻す

今回の PR は、この門番たち 自身 にあった「見逃し」「誤爆」「特定環境で動かない」という小さな不具合を直すもの。門番が少し賢く・頑丈になる、と思えばよい。

  • 取りこぼさない門番 — 大文字ファイル・一括編集・別コマンド経由など、これまですり抜けていた経路がまとめて塞がった

06おまけ — 今日その場で起きた「生きた実例」

皮肉なことに、この解説を書いている最中、口調チェックの門番 が、わらわが「禁止語を 説明のために引用しただけ」の文にも反応して、応答を一度差し戻した。門番は 「説明している人」と「違反している人」を見分けられぬ — これこそ今回の PR が向き合った根本原因 (人間の文章を機械的なパターンで読むことの限界) の、その場での生きた実演じゃった。文章を機械で読む限り、この種の誤爆は完全には無くせぬ構造的な宿題として残る。
