01なぜ「検証器が素通し」は怖いのか

three-elements-harness (3層ハーネス。AI エージェントを Macro=戦略層 と Micro=実装層 に分けて動かす仕組み) では、Macro 層が勝手に実装ファイルを書いたり下請けエージェントを直接呼んだりせぬよう、PreToolUse hook (ツール実行の直前に割り込む検査スクリプト) が見張っている。

問題は、その見張り役 2 本が fail-open (検査に失敗したのに「異常なし」として素通しさせてしまう挙動) を起こしていたこと。鍵が壊れているのに「施錠済み」ランプが点く金庫と同じで、守っているつもりで、実際には素通しになっていた。検証器そのもののバグは気づきにくいぶん、最も怖い。

02この 2 つの hook は何を守るのか

2 本はいずれも「いま Macro 層か?」を .docs/project-os/state/phase.yaml という小さな状態ファイルの phase: 値 (macro か micro の 2 値) で判定し、Macro のときだけ制限をかける。

  • block-team-in-macro.sh — Macro 層から下請けエージェント (team-* で始まる subagent) を直接呼ぶのを止める。本来は指揮 skill 経由で呼ぶ約束だから。
  • restrict-macro-writes.sh — Macro 層の書込先を .docs/ 配下のメモ類だけに限定する。実装ファイル (src/ 等) を Macro が直接書くのを止める。

どちらも判定の入口は同じ「phase.yaml を読む」一点。ここが 3 通りに抜けていた。

03壊れ方 ① — 「phase の読み方」が 2 通りに抜けていた

入口で phase 値を取り出すコードに、2 つのバグが同居していた。

バグ 2-1 (引用符): YAML では phase: macro も phase: "macro" も同じ意味。だが旧コードは引用符付きだと "macro" を引用符ごと取り、「macro という文字列ではない」と誤判定して素通しした。

バグ 2-2 (空 / 壊れ): phase.yaml が空や壊れていて値が取れないと、変数は空文字になる。旧コードは「macro 以外なら素通し」だったので、空っぽ=異常なのに素通し。コメントには「異常時は安全側に倒す (fail-closed)」と書いてあったのに、実装は真逆だった。

直し方: ① 取り出した値の前後の引用符を外す。② phase の正規値は macro か micro の 2 つだけなので、判定を「micro のときだけ素通し / それ以外 (macro・空・未知) は止める」に反転した。正規値が micro しかないので、正しい micro を誤って止める心配 (over-block) もない。

block-team-in-macro.sh / restrict-macro-writes.sh (要点抜粋) +2−1
4646 PHASE=$(awk '…' phase.yaml) # phase値を取り出す
47 − if [[ "$PHASE" != "macro" ]]; then # 引用符付き/空だと誤判定→素通し
47+ PHASE=引用符を除去した値 # " や ' を外す (bug 2-1)
48+ if [[ "$PHASE" == "micro" ]]; then # microのみ通す・空/未知はblock (bug 2-2)
4849 echo '{"continue": true}' # 素通し（許可）

04壊れ方 ② — .. で .docs の外へ書けた

restrict-macro-writes.sh は「書込先が .docs/logs/ 等で始まれば許可」と判定する。だが .docs/logs/../../../etc/passwd のように .. (1 つ上の階層へ戻る記号) を挟むと、文字列としては .docs/logs/ で始まるので許可されてしまう。実際には .docs の外、システム領域まで書ける抜け穴だった (パストラバーサル)。

直し方: 許可判定の前に、path の構成要素として .. を含むものは問答無用で止める。判定は /パス/ の形にして */../* と照合するので、v1..2.md のような「名前にたまたまドットが 2 つ並ぶ正規ファイル」は誤って止めない。

64+ case "/${NORMALIZED}/" in # 追加: 書込先に .. があれば
65+ */../* ) exit 2 ;; # 即 block (bug 2-3)
66+ esac
6467 case "$NORMALIZED" in
6568 .docs/logs/*) ALLOW=1 ;; # ../.. が紛れても通っていた
