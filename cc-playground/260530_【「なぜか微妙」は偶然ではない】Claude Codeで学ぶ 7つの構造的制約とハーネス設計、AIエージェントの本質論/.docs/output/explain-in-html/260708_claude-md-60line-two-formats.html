05共通で外してはならぬ3点

  • Tools 安全2行は inline 死守 — 未検証入力の直接注入禁止・prompt injection 防御は毎回効かせたい。`.docs/` へ逃がすと防御が遅延ロード化して薄まる。両案とも Prohibition に残す。
  • ポインタ先は `.docs/` であって `rules/` ではない — `rules/`(paths なし)は CLAUDE.md と同じく起動時に全注入される。そこへ移すと注入は1バイトも減らぬ。注入されない `.docs/progressive-disclosure/` が唯一の正解。
  • 壊れ検出は別途 — `hook_validate_claudemd.sh` は `rules/hooks/skills/references` は検証するが `.docs/` は見ぬ。ポインタの stale(リンク切れ)検出を足すなら随伴 issue が要る。
