01V章の主張 — 「気をつけてね」ではなくガードレール

Claude Code の出力は確率的で、指示の遵守も確率的。この二重の不確実性に「プロンプトの工夫」で対抗するのは根本的な間違いで、機械的に正誤を判定できるものはすべて決定論的ツールに任せよ — が章の背骨。たとえは「気をつけて運転してね」(注意力依存・疲れたら効かない) とガードレール (状態に無関係・100%防護)。判断基準は 1 行:「入力と正解の間に確定的な写像 (決まった答え) があるなら、LLM を挟む必要はない」。

note は再発の検知を人間の記憶に頼っている。ハーネスは発火台帳の再発センサー (recurrence、今月 535 行) が同種の違反の繰り返しを機械的に記録し、月次レポートで「注意書きが効いていない箇所」を数字で知らせる。昇格すべき場所が観測面から浮かび上がる。

超過3昇格ラダーの反 Goodhart 規律

「発火数が多い = 昇格すべき」と機械的に読むと、指標が目標になって壊れる (Goodhart の法則)。ハーネスは「多発火は上流の自然言語規律が綻んでいるサインであり、昇格シグナルと混同しない」と明文化し、昇格の最終判断を人間に残す — ラダーの運用が指標ハックに堕ちない設計。

超過4「LLM に任せるべきもの」の境界も明文化

note の分類表は「機械に任せるべきもの」を列挙する。ハーネスは逆側 —「内容の真偽 (意味論の判断) は機械化しない」— も rule に明記し、検証を人間 + Claude の規律に割り当てる根拠を残している。境界の両側が宣言済み。
