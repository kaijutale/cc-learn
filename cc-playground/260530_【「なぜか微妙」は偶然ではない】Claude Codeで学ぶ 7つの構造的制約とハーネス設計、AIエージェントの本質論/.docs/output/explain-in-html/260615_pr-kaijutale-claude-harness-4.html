1emoji 検知の移植性＋範囲穴

壊れていた

hook は grep -P（PCRE）で絵文字を探すが、bare grep は PATH 依存。macOS 標準の /usr/bin/grep（BSD grep）は -P 非対応で、エラーが 2>/dev/null に握り潰され、絵文字検知が静かにゼロになる（fail-open）。加えて Unicode 範囲が狭く、⭐ や c/r/tm 記号・国旗を取りこぼしていた。

こう直した

/usr/bin/perl（macOS/Linux 標準の絶対パス）に置換して PATH 非依存化。Unicode 範囲を拡張し、罫線素片（U+2500–257F）は範囲外に置いて誤検知を回避。-CSD で UTF-8 をデコード。

3essence-gate の 3 バグ一括（命名ドリフト＋散文誤読＋fail-open）

essence-gate は「skill/agent/hook を触る commit」を直近の essence レビュー verdict で gate する hook。ここに 3 つのバグが重なっていた。

  • (a) 命名ドリフト：glob *_self-eval-v*.md が最新命名（<date>_<slug>.md）を取りこぼし、古い self-eval を掴む。
  • (b) 散文の過去値誤読：verdict を散文 grep で抽出 → 本文中の過去値（例 "Critical 1 / High 3"）を誤って拾う。
  • (c) fail-open：新フォーマットで件数が取れないと 0/0 と仮定して approve（見逃し）。
こう直した

glob を *.md 化して最新命名に追従／機械可読フィールド critical_count・high_count（出力テンプレに既存）を行頭アンカーで優先抽出／取れなければ散文ペアに fallback／それも無ければ fail-closed（block）。orchestrator の出力契約は既にフィールドを持つため 契約変更なし（後方互換）。
