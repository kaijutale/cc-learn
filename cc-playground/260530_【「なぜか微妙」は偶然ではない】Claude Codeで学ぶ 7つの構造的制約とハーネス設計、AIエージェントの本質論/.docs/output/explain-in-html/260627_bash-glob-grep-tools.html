02Bash — コマンドを実行する

シェルコマンド(bash)を実行し、その出力を返すツールじゃ。git 操作・gh コマンド・ビルド・テスト・スクリプト実行など、「副作用のある決定論的な処理(実行すれば毎回同じ結果になる作業)」に使う。

  • 作業ディレクトリは保持される — 呼び出しをまたいで、移動した場所が維持される
  • timeout はミリ秒で指定 — 既定 120000(2分)、最大 600000(10分)
  • run_in_background(バックグラウンド実行) — 長時間かかるコマンドを裏で走らせられる
  • 避けるべき使い方 — cat / head / tail / sed / awk / echo / find / grep を Bash で代用しない(理由は後述。専用ツールを使う)

03Glob — ファイルを「名前・パス」で探す

glob パターン(ワイルドカードによるファイル名の指定)で、ファイルのパスを高速に探すツール。**/*.ts(全階層の .ts ファイル)や src/**/*.test.tsx のように書く。中身は見ない ── あくまでファイル名・パスのマッチングじゃ。

結果はマッチしたファイルパスの一覧(更新時刻順などでソート)。「この拡張子のファイル一覧が欲しい」「あのディレクトリ配下の特定パターンのファイルを知りたい」というときに使う。

04Grep — ファイルの「中身」を検索する

ファイルの中身を正規表現(文字列パターンの記法)で検索するツール。高速検索エンジン ripgrep がベースじゃ。「この関数はどこで使われている?」「この文字列を含むファイルは?」というときに使う。

主なパラメータ:

パラメータ意味
pattern検索する正規表現(必須)
path検索する場所(ファイル / ディレクトリ)
glob対象ファイルの絞り込み(例: *.ts)
type言語タイプで絞り込み(例: js, py)

-i / -n大文字小文字を無視 / 行番号を表示

05使い分け — 競合せず役割分担

やりたいこと使うツール
ファイルを名前・パスで探すGlob
ファイルの中身(テキスト)を探すGrep
コマンドを実行する(git / ビルド / 副作用)Bash
ファイルの中身を読む(参考)Read

比喩で言えば ── Glob はファイル名の地図、Grep は中身の検索エンジン、Bash は実行装置。3つは奪い合わず、それぞれの仕事を持つ。

06なぜ検索を Bash でやらないのか

Bash でも find(ファイル探し)や grep(中身検索)や cat(中身表示)は打てる。だが Claude Code は検索・読み取りを専用ツール(Glob / Grep / Read)に寄せる方針じゃ。理由は4つ:

  • 速度と最適化 — 専用ツールは検索に特化し、出力が読み手(Claude)向けに整形される
  • 権限・安全性 — 専用ツールは権限管理やサンドボックス(隔離された実行環境)と相性が良い
  • 移植性 — OS による差(macOS の BSD 系と Linux の GNU 系で find / grep の挙動が違う)に左右されない
  • 構造化された結果 — 出力が整理され、後の処理で扱いやすい

原則は ── 検索・読み取りは専用ツール、コマンド実行(副作用)は Bash という役割分担じゃ。

07総評

3つのツールは競合せず、役割で分担しておる ── Glob はファイル名の地図、Grep は中身の検索、Bash は実行装置。探すなら Glob / Grep、動かすなら Bash。find や grep を Bash で代用せず専用ツールに寄せるのが、速度・安全・移植性の面で Claude Code 流じゃ。
