03Critical 2 件 — 今この瞬間に効いている

CRITICAL絵文字チェックが macOS で完全に機能していない

対象hook_post_emoji_check.sh (ファイル編集後に絵文字混入を検査)
中身絵文字を探すのに grep -P(Perl 互換の正規表現)を使うが、macOS 標準の grep はこの -P を理解できず毎回エラーで止まる。そのエラーを 2>/dev/null で捨てているため、検査結果が常に空になる。
実害「コード内コメントに絵文字を入れない」という規約がまったく効いていない。失敗しても素通りさせる作り(fail-open)の最悪形で、何も検知しないまま「合格」を返し続ける。
# hook_post_emoji_check.sh  L9
EMOJI_LINES=$(grep -nP '[絵文字Unicode範囲]' "$FILE" 2>/dev/null)
              ^^^^^^^  macOS 既定の BSD grep は -P (Perl正規表現) 非対応
              → 毎回 "grep: invalid option -- P" でエラー終了
              → 2>/dev/null がエラーを隠し、EMOJI_LINES は常に空
結果: 絵文字付きコメントを 1 件も検知しない (検知ゼロ)

CRITICAL秘密ファイル検査が、安全なテンプレートまで誤って止める

対象hook_pre_read_secret_check.sh (秘密ファイルの読み取りを止める防御)
中身「.env」を止める正規表現が、値の入っていない見本ファイル .env.example や .env.template まで巻き込んで読み取り拒否する。
実害プロジェクトの設定見本(秘密情報を含まない)を参照できなくなる。CLAUDE.md L56「.env.example は読み書き可」と正面から矛盾する過剰ブロック。
# hook_pre_read_secret_check.sh  L42
/\.env(\.[^/]+)?$        ← .env と .env.<なんでも> に一致
照合 ".env.example"  →  (\.[^/]+)? が ".example" に一致  →  DENY (読取拒否)

だが CLAUDE.md L56:「.env.example (値なしテンプレ) は読み書き可」
→ 正当なテンプレ参照まで全停止する過剰ブロック

HIGH x63層構造の保護 hook が複数経路で破れる

対象block-team-in-macro.sh / restrict-macro-writes.sh (上位フェーズでの危険操作を禁じる)
引用符で失効フェーズ名を phase: "macro" と引用符付きで書くと、解析が引用符ごと読んでしまい「macro ではない」と誤判定。制限が丸ごと外れ、本来禁じられた操作が通る。
空ファイルで素通りフェーズ定義ファイルが空・壊れていると制限がスキップされる。スクリプト内コメントは「異常時は安全側に倒す」と謳うのに、実際は逆に通してしまう。
「..」で脱獄書込先を .docs/logs/../../../etc/passwd のように書くと許可パターンに一致し、保護領域の外へ書込めてしまう(通しすぎ)。

HIGH x3秘密・ハードコード検査の取りこぼし

ファイル名のみで抜ける秘密ファイル検査が先頭スラッシュ前提のため、id_rsa や .env をフルパスでなく名前だけで渡すと検知をすり抜ける(条件付き)。
.netrc 等も同様認証情報ファイル .netrc / .npmrc / .aws も同じ理由で相対パス形だと取りこぼす。
同一行に紛れて通過ハードコード検査が行単位で例外語を除外するため、1 行に許可語と実マシンパスが混在すると、その行ごと除外され実パスがすり抜ける。

HIGH x4コマンド検査・lint の穴

多語パターンの地雷hook_pre_commands.sh はシェル変数を素で展開するため、禁止コマンドに空白入りの語句を 1 つ足すと分断され、検査が崩れる(現状は未発火の潜在地雷)。
git のオプション付きを見逃すgenerated-comment-ban は git -C <dir> commit や git -c k=v commit を取りこぼし、禁止コメント文言が素通りする。
絵文字範囲の穴emoji 検査は(仮に grep が動いても)記号類 ☆ © ® 国旗などを範囲外として見逃す。
引用符付き設定を見逃すfork-disable-lint は context: "fork"(引用符)や末尾コメント付きの設定行を取りこぼし、矛盾を見逃す。

HIGH x4応答終了・セッション・圧縮時の検査

禁止語の部分一致で誤検知hook_stop_words.sh は語の境界を見ないため、「はず」が「はずれ」に、物理量の「半分」が、正当な「念のため」「一緒に」が禁止語として誤ブロックされる(6 例中 5 例で実証)。
状態ファイルの解析破壊handoff-check は、引き継ぎ状態ファイルの標準的な配列記法でYAML解析が壊れる。
未push件数が消えるセッション開始時の git 同期リマインダーが、リモート追跡情報が無い状況で未push件数を黙って 0 と表示する(見逃し)。
正常を「壊れ」と誤報告CLAUDE.md 検証が、説明文中の文字列を実パスと誤認し、正常な CLAUDE.md を圧縮のたびに「参照が壊れている」と誤報告する。

05横断する 5 つの構造的根因

個々のバグは別物に見えて、根は 5 つの設計上の癖に収束する。先行の essence-gate と同じ癖じゃ。

  • 1. 人間可読の文章を正規表現で読む — 禁止語の部分一致、CLAUDE.md の誤報告、設定行の取りこぼし、essence-gate の散文解析。書式が少し変わるだけで誤抽出・抽出失敗する。
  • 2. 失敗時に素通りさせる (fail-open) — 絵文字の grep 不発、空フェーズファイル、秘密ファイルのファイル名のみ、応答終了検査の見逃し。検査不能を「合格」と取り違える。
  • 3. 特定の命名・ツール名を決め打つ — 編集ツール MultiEdit に未対応(matcher が古く 3 hook が発火せず)、essence-gate のファイル名 glob、通知のサーバー名解析。命名が変わると追従できない。
  • 4. 境界の数え方を誤る — 行数カウントの 1 行ずれ(501 行取りこぼし)、区切りの閉じ忘れ見逃し、改行コード(CRLF)や先頭の見えない文字での誤判定。
  • 5. シェル変数を素で展開する — コマンド検査の語分断、禁止語ループ、書込先のワイルドカード誤展開。引用符を付けないことで意図せぬ分割・展開が起きる。

07推奨する対応順位

  • 1. 即修正 (Critical 2) — 絵文字検査の grep を perl/python 等に置換(検知ゼロの解消)、秘密ファイル検査に .env.example 系の除外を追加(CLAUDE.md との矛盾解消)。
  • 2. 早期 (3層保護の失効) — 引用符付きフェーズ・空フェーズファイル・「..」混じりパスの 3 経路を塞ぐ。Macro 層の最小権限が同時に破れている。
  • 3. 取りこぼし系 — 秘密/ハードコードの抜け道、git -C 付き commit の取りこぼし、禁止語の部分一致誤検知。
  • 4. 設計レベル — 5 つの根因を構造で断つ。文章解析は機械可読フィールドへ、素の変数展開は行単位読込へ、命名決め打ちは追従可能な形へ、文字列照合は語境界・アンカー付きへ。

08総評

これは essence-gate 単体の問題ではなく、ハーネスの hook 層全体に同じ設計の癖が広く根を張っている。「文章を正規表現で読む」「失敗時に素通りさせる」「変数を素で展開する」「命名を決め打つ」——この 4 つが 21 hook を横断する。個々の修正は小さいが、対症療法を積むより、根因 5 つを設計思想の統一で断つのが本筋だ。多くは潜在的・条件付きだが、絵文字検査と秘密ファイル検査の 2 つの Critical、そして 3層保護の失効は、今この瞬間に効いている。
