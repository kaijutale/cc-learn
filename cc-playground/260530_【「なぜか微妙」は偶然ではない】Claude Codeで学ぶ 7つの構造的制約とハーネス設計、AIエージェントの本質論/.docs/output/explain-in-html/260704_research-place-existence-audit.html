03なぜ「在るのに使われない」のか — 3つの構造要因

要因1「未使用 → 不可視 → 未使用」の自己強化ループ (実装コードで確定)

機構Claude Code は毎セッション、skill 一覧をコンテキストへ注入する際に合計文字数の上限を課す (既定: 約8,000字。v2.1.200 のコード解析で確定)。超過すると「使用回数 × 0.5^(最終使用からの日数 ÷ 7日)」という生存スコアの低い skill から説明文を削り、名前だけの表示に落とす。
帰結一度も使われていない skill は生存スコア0で最初に削られる。発動キーワードは説明文の中にあるので、削られた skill は自動発火の手段を失い、永遠に「未使用」のまま — ループが閉じる。
実測本セッションの一覧では約20本が名前だけの表示に落ちていた (conducting-research-phase を含む)。よく使う skill (logging 等) は生き残っていた。

要因2常駐文書への露出がゼロ

現状research の存在を伝える記述が、常時読み込まれる文書 (グローバル CLAUDE.md・rules・プロジェクト CLAUDE.md) に1行も無い。
