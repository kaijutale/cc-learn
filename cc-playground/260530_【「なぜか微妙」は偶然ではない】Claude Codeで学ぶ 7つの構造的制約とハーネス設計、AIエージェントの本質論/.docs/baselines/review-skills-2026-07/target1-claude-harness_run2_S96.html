03検出されたアンチパターン

WARNA2. ツール接続は帯域コストに見合っているか

検出事実MCP 5接続常駐。chrome-devtools・claude-in-chrome・context7 は代替不能で必然性明確。一方 brave-search は組込 WebSearch、firecrawl は組込 WebFetch と機能重複（CLAUDE.md で品質理由の明示選択）。
影響MCP のツール定義は毎セッション常駐。組込で代替可能な2接続分の帯域を、品質差というマージナルな利得のために払い続けている。実害は小さいが釣り合いに温度差。
関連原則(→C-1) コンテキスト帯域は有限でゼロサム。組込で賄える機能への追加ツール定義は帯域の純増。
改善案
brave-search / firecrawl の常駐価値を実測で棚卸し:
  - WebSearch/WebFetch への優位を体感できている → 現状維持で正当
  - 体感差が曖昧 → 2接続を外して組込に寄せると帯域が軽くなる
※ chrome-devtools / claude-in-chrome / context7 は据え置き推奨（代替不能）

WARNA4. エージェントの選択肢が不必要に広がっていないか

検出事実skill 90件が全て model-invocable。disable-model-invocation: true を持つ実 SKILL は2件のみ。fork/reviewer 型は description の「明示呼出専用」文言で抑制しており、構造的発火抑止ではない。permissions.allow は厳密スコープ済み（良好）。
影響90件の description が毎セッション「注文メニュー」に載る。fork/reviewer skill は本来 Skill ツールからの明示呼出専用だが、自動発火候補として推論リソースを消費し得る。
関連原則(→V-1.3) 選択肢は推論前に絞る, (→C-1.1) コンテキストの能動的管理。
改善案
# 各 fork / reviewer 系 skill の frontmatter に追加
# （明示 Skill(name) 呼出は維持。自動発火候補からのみ除外）
disable-model-invocation: true

# 対象: *-fork 系, essence-reviewing-orchestrator,

#       llm-debate-* fork 群, review-harness の内部 fork 等
