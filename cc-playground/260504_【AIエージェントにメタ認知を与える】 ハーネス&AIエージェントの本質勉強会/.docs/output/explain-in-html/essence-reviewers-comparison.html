Q2 · Single Reviewer

review-agent-essence の機能

構造 (単体 skill + subagent)

  • 4 step フロー (読込 → 関連原則選定 → 評価 → ハンドオフ)
  • context: fork で subagent (general-purpose, model: opus) を起動
  • 出力: 標準出力に Essence Review template
  • 修正 Plan 作成は 呼出元に委譲 (「次のアクション」ディレクティブ)
  • 並列度: なし (単体起動)

評価基準 (agent-essence.md 単一)

  • C-1〜C-7: LLM の構造的制約 (帯域有限 / 入力バイアス / 迎合性 / 自己申告無効 / 報酬ハッキング / 訓練データ断崖 / 校正盲)

  • = 11 原則 (サブ番号含むと 20+)
  • 「関連度判定」で原則選別 (機械的全件適用しない)

03
Q3 · Differential

2 つの skill の違い — 別系統の独立設計

構造的な違い

  • 単層 (subagent 1 体) vs 多層 (master + 3 sub + 3 agent = 7 ファイル)
  • 並列度なし vs 3 fork 並列
  • 4 step vs 9 step + 進捗 JSON 検証
  • stdout 出力 vs 永続化ファイル + Write tool
  • Lead 補完なし vs Step 3.5 で原則横断 root cause 統合
  • Plan 連携内蔵 vs 別 skill (accumulating-reviewer-feedback)

基準・出典の違い

  • agent-essence.md (11 原則、C/T/K/V/S/E カテゴリ) vs essentials.md × 3 (24 原則、harness/skill/ui 領域別)
  • 出典: Boris-Anthropic 系の Claude Code 設計文化 vs note 記事 page 12-15「レビューア = カスタムスキル」直接出典
  • 原則 ID 体系: C-1.1 vs harness-原則#1
関係性の核心

「essence」という共通語に騙されない。両者は 派生関係ではなく独立並走。design-rationale.md 核心 5 で「review-agent-essence は完全無変更、本 skill は新規追加で構造化、既存資産は触らない」と明示。Linux カーネルが BSD を取り込まずに独自実装し続けたのと同じ並走戦略。

関係性: Parallel (並存) 継承なし
04
Q4 · Verdict

優劣判定 — Orchestrator が多軸で優越

Orchestrator の優位 (7 軸)

  • カバレッジ: 24 原則 × 3 領域 (vs 11 原則)
  • 時間効率: 3 fork 並列で 1 領域あたり 30-40 秒 (vs 単体逐次)
  • 構造的盲点解消: Step 3.5 Lead 補完 (原則横断 root cause 統合)
  • 機械検証: 9 step validate-all-steps.sh で step skip 検出
  • 永続化フロー: 全 self-eval が ts 付きで蓄積 → 差分追跡可能
  • 反インフレ原則: 「全 🟢」を能動的に疑う構造
  • HITL 統合: Critical 検出時の人間チェックポイント

review-agent-essence の残存価値

  • 軽量・即時: 単体起動でコスト低 (3 fork は重い)
  • Plan 作成連携内蔵: 「次のアクション」ディレクティブで呼出元に YES/NO を投げる
  • 11 原則の質: Anthropic/Boris 系の概念 (C-1 ゼロサム / C-7 校正盲 等) が camone のメタ認知に直接寄与
  • 「触らず残す」運用: 改修禁止で並存、軽量レビューの receptacle
用途別の使い分け (camone 現運用)

包括的 self-eval + 品質階段運用 → essence-reviewing-orchestrator (現役主力、すべての Skill Creator 改善で使用)。軽量・即時レビュー + Plan 作成連携 → review-agent-essence (Legacy、現在は使用機会が限定的)。総合判定としては Orchestrator が優越するが、camone は両者を 並存させたまま運用している。

総合判定: Orchestrator 優越 (Strong) Verdict: Strong
