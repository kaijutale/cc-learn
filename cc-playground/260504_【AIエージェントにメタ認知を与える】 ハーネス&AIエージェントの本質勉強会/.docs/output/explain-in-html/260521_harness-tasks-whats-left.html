branch-validator の Hook 化（ブランチ名を作成時に検証）

ブランチ名の規則を、作成（git checkout -b）の瞬間に検証する Hook（PreToolUse）へ前倒しする案。検証ロジックは正規表現で決定論的に書ける（5 軸 4/5・実装は容易）。

着手前の判断

層選択の原則「被害が大きい時だけ Hook 化」に照らすと、ブランチ名ミスは被害が小さい。L2（CLAUDE.md ルール）維持で見送りも十分アリ。まず「やるべきか」から問う。

着手可 · 条件付

auditing-web-quality の fork 化（Web 品質監査を孫エージェントへ委譲）

Web ページ品質監査を context:fork（メインを汚さない独立コンテキスト）で回す案。ただし Chrome DevTools の連携（MCP）は合否を終了コードで返さず状態を持つので、合否を JSON 化する前処理の設計が先になる（5 軸 2-3/5・重め）。
