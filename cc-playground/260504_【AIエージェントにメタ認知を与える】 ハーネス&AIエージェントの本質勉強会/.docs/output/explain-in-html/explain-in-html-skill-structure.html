Why fixed CSS

5 軸 commitment と美学の固定

designing-beautiful-frontends skill は 毎回違う美学を要求する (anti-convergence)。しかし本 skill は 解説 skill であり、価値の核心は「一貫した読書体験」。両者の差異はタスク性質の差 — 新規プロダクト UI=多様性が価値、解説=ブランド一貫性が価値。

収束 (convergence) を防ぐのは 中間値の禁止。「ちょっと modern で…」「やや warm かな」と曖昧化した瞬間に Bootstrap になる。5 軸全部を一方の極に commit させることで、判断ブレを構造的に潰す。

AxisCommittedRationale
densitydense技術解説は情報密度が価値
formalityformaleditorial magazine の品格
eratimelessretro でも bleeding-edge でもない、書籍のような不時間性
warmthwarm銅 + 苔 + ivory の warm palette
color_intensitymutedvivid ではない、深みのある muted tone
Identity 固定こそが distinctiveness を保つ。 繰り返し使われる美学は、それ自体が十分に distinctive なら
AI slop ではなく ブランドになる。 — aesthetic-identity.md より
