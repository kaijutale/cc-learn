# 本質ドキュメントを構築するための各ステップ

step1. まず「自分のAIエージェントに何を評価させたいか」を1つ書き出してみよう。それが本質ドキュメントの出発点になる。

step2. 自分の領域（スキル設計、UI、コンテンツ等）で「誰が見ても正しいと言える原則」を3つ書き出してみよう。それが本質ドキュメントの種になる。

step3. 既存のスキルやワークフローに「レビューア呼び出し→フィードバック→再実行」の3ステップを1つ追加してみよう。

step4. 現在1つのエージェントに任せているタスクを「探索→計画→実装→レビュー」に分解し、最も効果の高い1つをサブエージェント化してみよう。

step5. 自分のワークフローで「毎回同じ結果が必要な処理」を1つ見つけ、スクリプトまたはHookに置き換えてみよう。

step6. 最もよく使うスキルのMarkdown末尾に `## Gotcha` セクションを追加し、過去に遭遇した失敗パターンを3つ書き出してみよう。
