機能名: cdd URLデコード関数

- セッション名: 未設定
- 日付: 2026-02-07 18:38:19
- 概要: Ghosttyで `file://` URL形式のパスを貼り付けた際にパーセントエンコーディングされた日本語パスを正しくデコードして `cd` するためのシェル関数を追加
- 実装内容:
  - `.zshrc` に `cdd` 関数を追加
  - 処理内容:
    1. `file://` プレフィックスを除去
    2. `%XX` 形式のパーセントエンコーディングを `\xXX` に変換
    3. `printf '%b'` でバイト列をUTF-8文字に復元
    4. デコード済みパスで `cd` を実行
  - 初回実装では `python3` を使用したが、引数が長すぎて `file name too long` エラーが発生したため、`printf` を使った純粋なシェル版に修正
- 設計意図:
  - Hyper（Electronベース）はクリップボードのURL自動デコードを行うが、Ghostty（ネイティブアプリ）はクリップボードの内容をそのまま貼り付けるため、この差異を吸収する関数が必要
  - `cd` を上書きせず新しいコマンド `cdd` として定義し、既存環境への影響をゼロにした
  - python3 依存を排除し、シェルビルトインのみで処理する軽量な実装に変更
- 副作用:
  - `cdd` という名前のコマンドと名前が衝突する可能性（実質ゼロリスク）
  - `zoxide` のフック `__zoxide_hook:2: command not found: zoxide` エラーが発生しているが、これは `cdd` とは無関係でPATHの読み込み順の問題
- 関連ファイル:
  - `~/.zshrc`（72-78行目付近）
