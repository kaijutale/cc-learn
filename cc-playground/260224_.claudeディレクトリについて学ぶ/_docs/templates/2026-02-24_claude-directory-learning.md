## 学んだこと

### 1. `.claude/` ディレクトリの役割

Claude Codeのプロジェクト固有の設定・メモリ・ルール・スキルを管理する中心地。

**3つのスコープ:**

| スコープ | 場所 | 対象 | チーム共有 |
|---------|------|------|----------|
| ユーザー | `~/.claude/` | 全プロジェクト共通 | ✗（個人） |
| プロジェクト | `.claude/` | このリポジトリのみ | ✓（git管理） |
| ローカル | `.claude/*.local.*` | このプロジェクト（個人） | ✗（.gitignore） |

### 2. CLAUDE.md vs CLAUDE.local.md の配置ルール（核心）

**CLAUDE.md（プロジェクトメモリ）:**
- `./CLAUDE.md` → 認識される
- `./.claude/CLAUDE.md` → 認識される
- **2箇所から選択可能**（機能は完全同一）

**CLAUDE.local.md（プロジェクトメモリ ローカル）:**
- `./CLAUDE.local.md` → 認識される
- `./.claude/CLAUDE.local.md` → **認識されない**
- **ルート直下の1箇所のみ有効**

### 3. 非対称性の理由（推測）

ローカル専用ファイル（CLAUDE.local.md）は、チーム共有前提の`.claude/`ディレクトリと混在させない設計思想と考えられる。

### 4. `.claude/` ディレクトリの全体構造

```
.claude/
├── CLAUDE.md              # プロジェクト共有指示
├── settings.json          # パーミッション・フック・環境変数
├── settings.local.json    # 個人プロジェクト設定（.gitignore）
├── rules/                 # モジュール化されたルール群
├── skills/                # カスタมスキル
├── agents/                # カスタムsubagents
├── commands/              # レガシーカスタムコマンド
└── hooks/                 # フック用スクリプト
```

### 5. メモリの読み込み順序

```
1. ~/.claude/CLAUDE.md          ← ユーザーグローバル
2. ~/.claude/rules/*.md         ← ユーザールール
3. ./.claude/CLAUDE.md          ← プロジェクト共有
   （または ./CLAUDE.md）
4. ./.claude/rules/*.md         ← プロジェクトルール
5. ./CLAUDE.local.md            ← 個人プロジェクト
6. ネストされた CLAUDE.md       ← サブディレクトリ（該当時のみ）
```

### 6. 公式ドキュメントのメモリタイプ一覧（5種類）

| メモリタイプ | ロケーション | 目的 | 共有対象 |
|---|---|---|---|
| エンタープライズポリシー | OS固有パス | IT/DevOps組織全体の指示 | 組織内全ユーザー |
| プロジェクトメモリ | `./CLAUDE.md` or `./.claude/CLAUDE.md` | チーム共有指示 | チームメンバー |
| プロジェクトルール | `./.claude/rules/*.md` | トピック固有の指示 | チームメンバー |
| ユーザーメモリ | `~/.claude/CLAUDE.md` | 個人設定 | 自分だけ |
| プロジェクトメモリ（ローカル） | `./CLAUDE.local.md` | 個人プロジェクト固有 | 自分だけ |

### 7. 注意点

- `.claude/CLAUDE.local.md` に誤配置するとClaude Codeは**黙って無視する**（エラーなし）
- CLAUDE.mdが500行を超えたら `rules/` への分割を検討
- `paths` フロントマターで条件付きルールロードが可能
