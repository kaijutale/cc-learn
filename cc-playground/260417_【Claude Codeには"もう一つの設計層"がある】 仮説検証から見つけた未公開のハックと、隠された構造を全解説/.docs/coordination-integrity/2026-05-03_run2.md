- **line**: 28
- **evidence**: `!\`ls package.json next.config.* tsconfig.json 2>/dev/null\``
- **description**:
  `^!` 行で `ls` に **glob 直書き** (`next.config.*`)。zsh の nomatch (errexit + nullglob無効) 環境では、対象ファイルが存在しない時に `zsh: no matches found: next.config.*` で hang する致命傷パターン。`2>/dev/null` で stderr を捨てても **コマンド自体が起動前に zsh の glob 展開段階で失敗する**ため副作用化不能。`ruleset C-3a` 典拠: `.docs/templates/2026-05-01_zsh-nullglob-result.md` で `ls openapi.*` パターンと同型を実証済。
- **修正方針** (本 audit skill は提案を出さないが、verifier 専念原則のため修正は呼出元責務):
  glob を `find -maxdepth 1` に置換 or `next.config.js` `next.config.mjs` `next.config.ts` を literal 列挙、または `2>/dev/null` を削って fallback パターンに `|| echo` を追加。
