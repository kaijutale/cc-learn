機能名: Claude Code 入力モードの呼称整理（bash mode / memory mode / 通常）

- セッション名: （未設定）
- 日付: 2026-04-18 15:52:24
- 概要: Claude Code の `!` プレフィックスの正式な呼称を確認し、入力モードの体系を整理した学習セッション。わたしが最初「bangコマンド」と記号ベースで誤回答し、かもねから「bashじゃないの？」と訂正された。
- 実装内容:
  - `!` プレフィックスの機能名は **bashコマンド（bash mode）** が Claude Code 公式の正式呼称であることを確定
  - Claude Code の3入力モードを整理:
    1. 通常モード（プロンプト入力）
    2. `!` → bash mode（現在のシェル環境でコマンド実行、出力が会話に取り込まれる）
    3. `#` → memory mode（CLAUDE.md への追記）
  - 記号の読み（bang / exclamation / ビックリマーク）と機能名（bashコマンド）を分離して理解
- 設計意図:
  - 記号名と機能名を混同しない習慣を定着させる。類例: `/` は slash だが機能名は「slashコマンド」、`@` は at だが「mention」
  - bash mode はサブシェルではなく**現在のセッションのシェル環境**で実行される点を押さえる（cd 後のディレクトリや環境変数が引き継がれる）
  - 学習ログに「誤回答 → 訂正」のプロセスを残すことで、同じ誤りを将来繰り返さない
- 副作用:
  - わたしの初回応答で「bangコマンド」と記号ベースの通称で回答したのは、Claude Code 機能名としては**不正確**だった。今後 `!` の機能を指すときは「bashコマンド」「bash mode」を第一選択とする
  - `logging-implementation` スキルのテンプレートは「実装」前提の項目名（実装内容/設計意図/副作用）だが、学習記録にも適応可能であることを確認（かもねの指摘により判明）。わたしがスキル適用範囲を過度に狭めていた
