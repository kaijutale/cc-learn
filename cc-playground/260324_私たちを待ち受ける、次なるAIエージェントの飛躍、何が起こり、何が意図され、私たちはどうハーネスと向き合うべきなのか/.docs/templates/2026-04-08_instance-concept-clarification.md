- 実装内容:
  - **かもねの仮説の検証**: 「LLM = インスタンス？」→ No。LLMはインスタンスの一部品であり、インスタンスそのものではない
  - **用語体系の整理**:
    - LLM = 素のモデル（能力のテンプレート）
    - エージェント = LLM + ツール + ルール + フィードバックループ
    - インスタンス = エージェントの実行単位（独立したコンテキストを持つ）
    - ハーネス = エージェントが力を発揮するための構造設計
  - **インスタンスの3点セット定義**: LLM + ハーネス構造 + 独立したコンテキスト
  - **DQアナロジー**: DQ3ルイーダの酒場システムでマッピング
    - LLM → 職業の知識体系（戦士の剣術、魔法使いの呪文理論）
    - インスタンス → 酒場で実際に作成された個々のキャラクター
    - ハーネス → 装備 + スキル構成 + 作戦指示
    - 並列インスタンス → 同じ職業のキャラを複数作り、別ダンジョンに同時派遣
  - **Peter Steinbergerの事例との接続**: tmuxで複数インスタンスを並列実行 = 複数キャラを別ダンジョンに同時派遣
- 設計意図:
  - LLMとインスタンスの混同は、エージェント設計の理解を妨げる根本的な誤解になりうるため、早期に正確な区別を確立した
  - DQアナロジーにより「職業 ≠ キャラクター」という直感的な理解を得ることで、「LLM ≠ インスタンス」が自然に腹落ちする構造にした
  - 前回セッション（RL概念マッピング）で確立した「ハーネス = 状態空間 + 行動空間 + 報酬の上位概念」と矛盾なく接続。インスタンスはそのハーネス構造の中で実際に動く実行単位

## 学び

- **同じLLMから複数インスタンスが生まれる**: DQ3で同じ「戦士」職から「ライアン」「ハッサン」が独立して存在できるように、同じClaude Opusから装備（ハーネス）の異なる複数インスタンスが並列稼働できる
- **インスタンスの独立性 = 注意力の独立性**: 各キャラのHP/MPが独立しているように、各インスタンスのコンテキスト（注意力）が独立しているからこそ「注意力のパイ」が各100%になる。これが記事の3本柱の第1柱「関心ごとの分離」の実装基盤
- **前回のRL概念との統合**: インスタンスは「RL環境（ハーネス）の中で実際に行動するエージェント」に相当。ハーネスが環境を設計し、インスタンスがその中で動く
