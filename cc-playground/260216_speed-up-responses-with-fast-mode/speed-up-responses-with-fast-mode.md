# ファストモードでレスポンスを高速化する

**公開日:** 2026年02月16日

**原文:** [Speed up responses with fast mode](https://code.claude.com/docs/en/fast-mode)

---

ファストモードは[リサーチプレビュー](https://code.claude.com/docs/en/fast-mode#research-preview)段階です。機能、料金、提供状況はフィードバックに基づいて変更される可能性があります。

ファストモードはClaude Opus 4.6の高速設定で、モデルを2.5倍高速にする代わりに、トークンあたりのコストが高くなります。インタラクティブな作業（高速なイテレーションやライブデバッグなど）でスピードが必要なときに `/fast` でオンに切り替え、レイテンシよりもコストが重要なときはオフに切り替えてください。

ファストモードは別のモデルではありません。同じOpus 4.6を、コスト効率よりもスピードを優先する異なるAPI設定で使用します。品質と機能はまったく同じで、レスポンスが速くなるだけです。

知っておくべきこと：

- Claude Code CLIで `/fast` を使ってファストモードのオン/オフを切り替えます。Claude Code VS Code Extensionでも `/fast` で利用可能です。
- Opus 4.6でのファストモードの料金は $30/150 MTok からです。ファストモードは、2月16日太平洋時間午後11時59分まで、すべてのプランで50%割引で利用可能です。
- サブスクリプションプラン（Pro/Max/Team/Enterprise）およびClaude Consoleのすべてのユーザーが利用できます。
- サブスクリプションプラン（Pro/Max/Team/Enterprise）のClaude Codeユーザーの場合、ファストモードはエクストラユーセージ（追加利用分）のみで利用可能であり、サブスクリプションのレート制限には含まれません。

このページでは、[ファストモードの切り替え方法](#toggle-fast-mode)、[コストのトレードオフ](#understand-the-cost-tradeoff)、[使用すべきタイミング](#decide-when-to-use-fast-mode)、[要件](#requirements)、[レート制限の動作](#handle-rate-limits)について説明します。

## ファストモードの切り替え

ファストモードは以下のいずれかの方法で切り替えられます：

- `/fast` と入力して Tab を押すとオン/オフが切り替わります
- [ユーザー設定ファイル](https://code.claude.com/docs/en/settings)で `"fastMode": true` を設定します

ファストモードはセッション間で維持されます。最もコスト効率を高めるには、会話の途中で切り替えるのではなく、セッション開始時にファストモードを有効にしてください。詳細は[コストのトレードオフを理解する](#understand-the-cost-tradeoff)を参照してください。

ファストモードを有効にすると：

- 別のモデルを使用中の場合、Claude Codeは自動的にOpus 4.6に切り替わります
- 確認メッセージ「Fast mode ON」が表示されます
- ファストモードがアクティブな間、プロンプトの横に小さな `↯` アイコンが表示されます
- いつでも `/fast` を再度実行して、ファストモードがオンかオフかを確認できます

`/fast` を再度実行してファストモードを無効にしても、Opus 4.6のままです。以前のモデルには戻りません。別のモデルに切り替えるには `/model` を使用してください。

## コストのトレードオフを理解する

ファストモードは標準のOpus 4.6よりもトークンあたりの料金が高くなります：

| モード | 入力（MTok） | 出力（MTok） |
| --- | --- | --- |
| Opus 4.6でのファストモード（200K未満） | $30 | $150 |
| Opus 4.6でのファストモード（200K超） | $60 | $225 |

ファストモードは100万トークンの拡張コンテキストウィンドウと互換性があります。

会話の途中でファストモードに切り替えると、会話コンテキスト全体に対してファストモードの未キャッシュ入力トークン料金が全額課金されます。これは、最初からファストモードを有効にしていた場合よりもコストが高くなります。

## ファストモードを使うべきタイミング

ファストモードは、コストよりもレスポンスのレイテンシが重要なインタラクティブな作業に最適です：

- コード変更の高速イテレーション
- ライブデバッグセッション
- 締め切りが厳しい時間重視の作業

標準モードが適しているケース：

- スピードがそれほど重要でない長時間の自律タスク
- バッチ処理やCI/CDパイプライン
- コスト重視のワークロード

### ファストモード vs エフォートレベル

ファストモードとエフォートレベルはどちらもレスポンス速度に影響しますが、仕組みが異なります：

| 設定 | 効果 |
| --- | --- |
| **ファストモード** | モデルの品質は同じ、レイテンシが低い、コストが高い |
| **低エフォートレベル** | 思考時間が短い、レスポンスが速い、複雑なタスクでは品質が低下する可能性がある |

両方を組み合わせることもできます：簡単なタスクで最大速度を得るために、ファストモードと低い[エフォートレベル](https://code.claude.com/docs/en/model-config#adjust-effort-level)を併用できます。

## 要件

ファストモードには以下のすべてが必要です：

- **サードパーティクラウドプロバイダーでは利用不可**：ファストモードはAmazon Bedrock、Google Vertex AI、Microsoft Azure Foundryでは利用できません。ファストモードはAnthropic Console APIおよびエクストラユーセージを使用するClaudeサブスクリプションプランで利用可能です。
- **エクストラユーセージの有効化**：アカウントでエクストラユーセージ（プランの含有分を超えた課金を許可する機能）が有効になっている必要があります。個人アカウントの場合は、[Consoleの課金設定](https://platform.claude.com/settings/organization/billing)で有効にしてください。TeamsおよびEnterpriseの場合は、管理者が組織のエクストラユーセージを有効にする必要があります。

ファストモードの使用量はエクストラユーセージに直接課金されます。プランの含有分が残っていても同様です。つまり、ファストモードのトークンはプランの含有使用量にカウントされず、最初のトークンからファストモード料金で課金されます。

- **TeamsおよびEnterpriseでの管理者による有効化**：ファストモードはTeamsおよびEnterprise組織ではデフォルトで無効です。管理者がユーザーがアクセスできるように、明示的に[ファストモードを有効化](#enable-fast-mode-for-your-organization)する必要があります。

管理者が組織でファストモードを有効にしていない場合、`/fast` コマンドは「Fast mode has been disabled by your organization.」と表示されます。

### 組織でファストモードを有効にする

管理者は以下の場所でファストモードを有効にできます：

- **Console**（APIカスタマー）：[Claude Codeの設定](https://platform.claude.com/claude-code/preferences)
- **Claude AI**（TeamsおよびEnterprise）：[管理者設定 > Claude Code](https://claude.ai/admin-settings/claude-code)

## レート制限の処理

ファストモードには標準のOpus 4.6とは別のレート制限があります。ファストモードのレート制限に達するか、エクストラユーセージのクレジットがなくなると：

1. ファストモードは自動的に標準のOpus 4.6にフォールバックします
2. `↯` アイコンがグレーになり、クールダウン中であることを示します
3. 標準の速度と料金で作業を続けられます
4. クールダウンが終了すると、ファストモードが自動的に再有効化されます

クールダウンを待つ代わりに手動でファストモードを無効にするには、`/fast` を再度実行してください。

## リサーチプレビュー

ファストモードはリサーチプレビュー機能です。これは以下を意味します：

- 機能はフィードバックに基づいて変更される可能性があります
- 提供状況と料金は変更される可能性があります
- 基盤となるAPI設定は進化する可能性があります

問題やフィードバックは、通常のAnthropicサポートチャネルを通じてご報告ください。
