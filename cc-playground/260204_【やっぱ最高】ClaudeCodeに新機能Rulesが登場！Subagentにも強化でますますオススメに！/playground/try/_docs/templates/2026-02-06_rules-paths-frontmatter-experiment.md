機能名: Rules paths フロントマター動作検証

- セッション名: なし
- 日付: 2026-02-06 22:57:16
- 概要: Claude Code Rules の `paths` フロントマターによるルール発動条件の挙動を実験・検証した
- 実装内容:
  - README.md を編集し、`# こんにちは` セクションを追加
  - この編集時に、以下の2つのルールの発動状況を観察:
    1. `general-rule.md`（pathsなし）→ 発動した（「わんわん」と発言）
    2. `ts-rule.md`（paths: `src/**/*.ts`）→ 発動しなかった（「にゃーん」なし）
- 設計意図: `paths` フロントマターの有無によるルール適用範囲の違いを実証するための実験
- 副作用: なし
- 関連ファイル:
  - .claude/rules/general-rule.md（pathsなし: 全ファイル適用）
  - .claude/rules/ts-rule.md（paths: src/**/*.ts のみ適用）
  - .claude/CLAUDE.md（@.claude/rules/ts-rule.md を参照）
  - README.md（編集対象ファイル）

## 実験結果まとめ

| ルール | paths 設定 | 編集ファイル | 発動 |
|--------|-----------|-------------|------|
| general-rule.md | なし（全ファイル対象） | README.md | した |
| ts-rule.md | `src/**/*.ts` | README.md | しなかった |

## 得られた知見

1. `paths` フロントマターが設定されたルールは、マッチするファイルを扱う時のみコンテキストに読み込まれる
2. `CLAUDE.md` の `@` 参照はドキュメントとしての参照であり、ルールの強制発動には影響しない
3. `paths` なしのルールは全ファイルに対して常に適用される
4. この仕組みにより、不要なルールをコンテキストに載せずに済み、コンテキストウィンドウの節約になる
