# コメント・ドキュメント規約

このルールは全ファイルに適用される。

## 基本方針

- コードは「何をしているか」を自明にし、コメントは「なぜそうしているか」を説明する
- 不要なコメントは書かない（コードを読めばわかることは書かない）

## JSDoc コメント

公開API（export された関数・クラス）には JSDoc を付ける。

```typescript
/**
 * ユーザー情報を取得する
 * @param userId - 取得対象のユーザーID
 * @returns ユーザー情報。存在しない場合は null
 */
export function getUser(userId: string): User | null {
  // ...
}
```

## TODO コメント

未完了のタスクには TODO コメントを使用。

```typescript
// TODO: エラーハンドリングを追加する
// TODO(camone): パフォーマンス改善が必要
```

## 禁止事項

- コメントアウトしたコードを残さない（Git履歴で管理）
- 絵文字をコメントに含めない
- 「ここを修正」などの曖昧なコメントは禁止
